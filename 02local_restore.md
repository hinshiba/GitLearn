## ローカルでの復元

コミットを変更の保存と説明したように，コミットした状態まで巻き戻す方法が存在する．
それぞれに特徴があるため，よく理解してほしい．

登場するコマンド
- `git restore`
- `git revert`
- `git reset`

これらは「何を」「どこまで」戻すかが異なる．
大まかには，`git restore`はまだコミットしていない変更を戻すもの，`git revert`と`git reset`はコミット済みの変更を戻すものである．

### `git restore`

`git restore`は，まだコミットしていない変更を元に戻すコマンドである．
コミット履歴は変更しないため，最も気軽に使える．

ワーキングツリーで編集したものの，その変更を破棄して直前のコミットの状態に戻したいときは，対象のファイルを指定する．

```bash
git restore <ファイル名>
```

すでに`git add`でステージングしてしまった変更を，ステージングエリアから取り下げたい(ワーキングツリーの変更自体は残す)ときは，`--staged`を付ける．

```bash
git restore --staged <ファイル名>
```

VSCodeでは，ソース管理ビューで変更されたファイルにカーソルを合わせると現れる「変更を破棄」(矢印が回転したアイコン)から，コミット前の変更を破棄できる．
これは`git restore`に相当する操作である．

破棄した変更は元に戻せないことが多いため，実行前に本当に不要な変更かを確認すること．

### `git revert`

`git revert`は，指定したコミットの変更を打ち消す新しいコミットを作成するコマンドである．

```bash
git revert <コミット>
```

過去のコミットそのものを消すのではなく，その変更を逆に適用したコミットを履歴の先頭に追加する点が特徴である．
履歴を書き換えないため，すでに他の人と共有した(pushした)変更を取り消す場合でも安全に使える．

### `git reset`

`git reset`は，現在のブランチの先端を指定したコミットまで移動させるコマンドである．
移動先より後のコミットは履歴から外れることになる．

```bash
git reset --mixed <コミット>
```

どこまで戻すかをオプションで指定する．

| オプション | コミット履歴 | ステージングエリア | ワーキングツリー |
| --- | --- | --- | --- |
| `--soft` | 戻す | そのまま | そのまま |
| `--mixed`(既定) | 戻す | 戻す | そのまま |
| `--hard` | 戻す | 戻す | 戻す |

`--soft`は，コミットだけを取り消して変更内容はステージングエリアに残すため，コミットし直したいときに使う．
`--mixed`(オプションを省略したときの既定)は，ステージングも解除して変更をワーキングツリーに残す．
`--hard`は，ワーキングツリーの変更まで含めてすべて戻すため，編集内容が失われる点に注意する．

`git reset`は履歴を書き換えるため，すでに共有した(pushした)コミットに対しては使わないこと．
共有済みの変更を取り消したい場合は`git revert`を用いる．

### 使い分け

- まだコミットしていない作業中の変更を捨てたい → `git restore`
- 共有済みのコミットを安全に取り消したい → `git revert`
- まだ共有していないローカルの履歴を整理したい → `git reset`
